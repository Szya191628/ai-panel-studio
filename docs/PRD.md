# AI Panel Studio — 产品需求文档 (PRD)

## 1. 产品定位

**AI Panel Studio** 是一款"AI圆桌讨论"本地Web应用：用户输入任意讨论话题并指定参会专家人数，系统调用大模型动态生成一组主持人与专家嘉宾，展示每位嘉宾的姓名、职业、Title、立场与专属颜色标识；用户确认阵容后进入演播厅讨论模式，观看一场由AI驱动、实时推进的圆桌讨论。

**核心价值主张**：让任何人都能瞬间召集一支"虚拟智库"，围绕任意议题展开深度碰撞。

## 2. 用户画像

| 用户群体 | 特征 | 核心诉求 |
|---------|------|---------|
| 知识工作者与研究人员 | 需要快速多角度审视议题 | 几分钟内获得不同视角的深度分析 |
| 产品/决策人员 | 方案评审前预演各方立场 | 发现盲点、检验假设、提升决策质量 |
| 教育与内容创作者 | 需要结构化呈现复杂议题 | 获得可录制、可回放的"虚拟专家访谈" |

## 3. 功能需求

### 3.1 首页

- 展示当前所有讨论列表（卡片式）
- 每张卡片显示：话题、状态、专家人数、创建时间
- "新建讨论"按钮 → 弹窗输入话题+专家人数
- 点击卡片进入对应演播厅

### 3.2 嘉宾生成

- 用户输入话题与专家人数后，系统调用大模型动态生成主持人+专家阵容
- 展示每位嘉宾：姓名、职业/Title、立场、专属颜色标识
- 支持"重新生成"操作
- 用户确认后进入演播厅

### 3.3 演播厅模式

**布局设计**：
- 左侧面板：嘉宾状态小窗（头像、姓名、Title、状态指示灯、关注点）
- 中间区域：Transcript（发言人气泡+色块标识）
- 右侧面板：共识/分歧实时更新区

**讨论引擎**：
- 主持人负责开场、追问、串联和总结
- 专家根据当前transcript自主决定发言顺序（举手/抢答/补充/反驳）
- 每次发言控制在1-3句
- 不允许机械式轮流发言

**实时更新**：
- 使用SSE推送发言、状态变更、共识/分歧
- 各区域独立滚动，不依赖页面滚动

### 3.4 嘉宾状态小窗

- 每位专家有独立小窗
- 实时显示Agent运行状态：待机/准备发言/发言中
- 显示当前关注点或公开思考摘要
- 不展示真实的隐藏chain-of-thought

### 3.5 实时共识与分歧

- 讨论过程中持续从发言中提炼共识与分歧
- 实时更新对应区域
- 不等到讨论结束后才生成

### 3.6 总结

- 结束时主持人以自然语言进行总结
- 禁止将JSON原文显示到页面上
- 结论弹窗展示最终总结

## 4. 非功能需求

| 需求 | 说明 |
|------|------|
| 本地运行 | 前后端分离，可本地启动 |
| API Key安全 | 大模型API Key只放后端环境变量 |
| 多讨论并行 | 不同讨论状态、事件流、transcript互相隔离 |
| 实时更新 | SSE推送 |
| 中文UI | 页面为中文界面 |
| 响应式布局 | 超宽屏、普通桌面与窄屏均需合理分配 |

## 5. 数据模型

### Discussion（讨论）
- id: 唯一标识
- topic: 讨论话题
- status: configuring / active / concluded
- expert_count: 专家人数
- created_at: 创建时间
- concluded_at: 结束时间
- conclusion: 结论

### Panelist（嘉宾）
- id: 唯一标识
- discussion_id: 所属讨论
- name: 姓名
- title: 职业/Title
- stance: 立场
- color: 专属颜色
- role: host / expert
- status: idle / preparing / speaking
- focus: 当前关注点

### Speech（发言）
- id: 唯一标识
- discussion_id: 所属讨论
- panelist_id: 发言嘉宾
- content: 发言内容
- speech_type: open / comment / rebut / question / summary
- round_num: 轮次

### Finding（共识/分歧）
- id: 唯一标识
- discussion_id: 所属讨论
- type: consensus / disagreement
- content: 内容
- round_num: 轮次
